Getting Started: Running the “PIC32MX USB Starter Kit - Generic Device” demo

 

Required Hardware

Firmware

Running the Demo

 

Required Hardware:

  To run this project, you will need one of the following sets of hardware:

 

   PIC32MX USB Starter Board (DM320003)

 

Firmware:

To run this project, you will need to load the corresponding firmware into the device.

 

The source code for this demo is available in the <Install Directory>\usb_sk_generic_device_demo directory.  In this directory you will find all of the user level source and header files as well as project and workspace files for each of the hardware platforms.  Find the project (*.mcp) or workspace (*.mcw) file that corresponds to the hardware platform you wish to test.  Compile and program the demo code into the hardware platform.  For more help on how to compile and program projects, please refer to the MPLAB® help available through the help menu of MPLAB (Help->Topics…->MPLAB IDE).

 

 

Running the Demo:

A PC can act as the (AKA Custom) USB host and this demo will run as the USB peripheral. 

 

Program the peripheral with the firmware found in the <Install Directory>\usb_sk_generic_device_demo folder.

 

Launch the host application on the PC, PDFSUSB.exe program found in the <Install Directory>\USB Tools\Pdfsusb directory.  This host application will display the temperature and potentiometer data when the “Demo Mode” tab is selected and the device has been selected and connected.

 

The first time the demo is connected to a Windows Host PC, you will need to access the MCHPUSB driver.  Point the Windows driver installer to “mchpusb.inf” located in <Install Directory>\USB Tools\MCHPUSB Custom Driver\MCHPUSB Driver\Release.

 

How to use:

1. Connect to Generic (AKA Custom) Host.  (LED 1 will blink.)

2. Press SW2 to increase Potentiometer (POT) value.

3. Press SW2 + SW1 to decrease Potentiometer (POT) value.

4. Press SW3 to increase temperature value.

5. Press SW3 + SW1 to decrease temperature value.
