Required Hardware:

  To run this project, you will need one of the following sets of hardware:

 

    Configuration: Explorer 16

Explorer 16 (DM240001)

            USB PICtail™ Plus Daughter Card (AC164131)

PIC32MX USB Plug-In-Module (PIM) (MA320002)

 

Configuring the Hardware:

 

This section describes how to set up the various configurations of hardware to run this demo.

Configuration: Explorer 16

 

Configuration: Explorer 16

1) Before attaching the PIM to the Explorer 16 board, insure that the processor selector switch (S2) is in the “PIM” position as seen in the image below.

 

2) Short the J7 jumper to the “PIC24” setting

 

3) Before connecting the PIM to the Explorer 16 board, remove all attached cables from both boards.  Connect the PIM to the Explorer 16 board.  Be careful when connecting the boards to insure that no pins are bent or damaged during the process.  Also insure that the PIM is not shifted in any direction and that all of the headers are properly aligned.

 

4)  On the USB PICTail Plus board, short jumper JP1.  Remove all other shorts on the board.

 

5)  Connect the USB PICTail Plus board to either of the female PICTail Plus connectors or on the card edge connector (J9) at the edge of the Explorer 16 board.

 

 

Firmware:

To run this project, you will need to load the corresponding firmware into the devices.  There are two methods available for loading the demos: Precompiled demos and source code projects.

 

Precompiled Demos are available in the “<Install Directory>\USB Precompiled Demos” folders.  Each demo should be prefaced with the hardware platform it is compiled for.  Select the file that matches the hardware that you have and the demo that you want to run.  For more information about how to load a precompiled project, please see the “Getting Started – Loading a precompiled demo” guide.

 

The source code for this demo is available in the “<Install Directory\usb_cdc_serial_device_demo” directory.  In this directory you will find all of the user level source and header files as well as project and workspace files.  Find the project (*.mcp) or workspace (*.mcw) file.  Compile and program the demo code into the hardware platform.  For more help on how to compile and program projects, please refer to the MPLAB® IDE help available through the help menu of MPLAB IDE (Help->Topics…->MPLAB IDE).

 

Please note you will be required to select the correct processor for the demo board.

1) Open the associated project file

            2) In MPLAB Select “Configure->Select Device”

            3) In the device list box in the top left corner of that window, select the PIC32MX460F512L device.

 

 

Running the Demo:

This demo allows the device to appear like a serial (COM) port to the host.  This demo will take data sent over the USB CDC interface and send it on the UART of the microcontroller.

 

In order to run this demo first compile and program the target device.  Attach the device to the host with the USB cable.  If the host is a PC and this is the first time you have plugged this device into the computer then you may be asked for a .inf file. 

 

 

Select the “Install from a list or specific location (Advanced)” option.   Point to the “<Install Directory>\USB Tools\USB CDC Serial Demo\inf” directory

 

 

Once the device is successfully installed, open up a terminal program, such as hyperterminal.  Select the appropriate COM port for the USB virtual COM port.  (This will be the new COM port that was not available before the installation.)

 

Once everything is configured correctly, anything typed into the terminal window will be echoed back by the device.  (Be sure that you haven’t enabled “local echo” or anything typed will be duplicated.)  Also, pressing S3 on the Explorer 16 board will cause the device to display a “Hello” message on the terminal window.

 

Note: Some terminal programs, like hyperterminal, require users to click the disconnect button before removing the device from the computer.  Failing to do so may result in having to close and open the program again in order to reconnect to the device.
