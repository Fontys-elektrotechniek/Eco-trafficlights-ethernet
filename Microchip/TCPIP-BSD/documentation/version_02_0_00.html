Changes

    The following new features have been added/modified to the current release of the MCHP BSD:
·            There is only one file that the user of the stack has to include now: “<Install Directory>\Microchip Solutions\Microchip\include\TCPIP-BSD\tcpip_bsd.h”. This one sub-includes all the necessary files the stack uses.
·            The user needs to modify/update the project “source/tcpip_bsd_config.h” file instead of eTCP.def. This file contains all the parameters needed for the stack configuration.
·            The source code of the stack has been moved to: “<Install Directory>\Microchip Solutions\Microchip\TCPIP-BSD\” directory.
The old extra “\source” directory was removed so that is more consistent with all distributed Microchip software.
Please update your projects accordingly.
·            Integration of the PIC32MX6 and PIC32MX7 families with the integrated Ethernet/MAC controller.
·            The stack works now in interrupts and presents events to the user application.
The application has to call into the stack only when Ethernet events are pending. This feature facilitates easy
integration in an OS environment where different RX/TX server/client threads have to be maintained.
See the tcpip_bsd_control.h for complete details
·            More consistent naming applied to the stack interface functions:
o             The BSD sockets interface file has been renamed socket.h (instead of sockapi.h).
o             All the proprietary stack related functions start now with the TCPIP letters (part of tcpip_bsd_control.h)
1.                          int TCPIPInit(unsigned int sys_clk) -- replaced -- void InitStackMgr(void)
2.                          int TCPIPProcess(void) -- replaced -- void StackMgrProcess()
3.                          TCPIPSetDefaultAddr() takes now explicit parameters to set the host and gateway IP address, the network mask and the host MAC address.
4.                          TCPIPSetIPAddr() takes a 32 bit IP adrress as parameter. TCPIPSetIPAddrByte() has been added to take 4 byte parameters.
5.                          TCPIPSetIPMask(),TCPIPSetIPGateway() have been renamed TCPIPSetIPMaskByte(),TCPIPSetIPGatewayByte() to be consistent with the new naming convention
6.                          New dynamic configuration functions have been added: TCPIPClose(),TCPIPSetRxProcessLimit(),TCPIPSetBuffMgrHeapSize(),TCPIPSetSocketsNo(),TCPIPSetMaxAckQueue().
7.                          See the tcpip_bsd_control.h for complete details.
·            The memory allocation has been improved: it is much faster and with smaller granularity.
·            TCP Keep Alive functionality has been added for stream sockets.
·            The stack supports now the Zero Copy functionality (much faster than the standard socket interface).
This feature is available only on the PIC32MX6-7 families with embedded Ethernet controller.
·            Support for external RMII/MII connected PHY's has been added to the stack.
Currently we support the National DP83848 and SMSC LAN 8700 PHY's. Support for other PHY's will be added in the future.
·            PHY scan operations are supported for PHY's status. This increases the speed of the link down status detection.
·            The TCP timer needed for the TCP state machine uses a global system timer which is implemented by the CPU core timer.
The peripheral Timer1 is no longer used.
 
Limitations of the MCHP BSD socket interface implementation:
·                The MCHP BSD TCP/IP stack is a subset of a standard, large TCP/IP stack implementation.
Some of the most notable limitations are:
o                     setsockopt(): the only options supported are TCP_NO_DELAY, SO_SNDBUF and SO_RCVBUF
o                     SO_RCVBUF is used for the socket window size advertisement.
All the traffic for all the sockets is stored in a global heap under the control of the stack manager.
o                     getsockopt() is not supported
o                     Other advanced options (like delayed acknowledge, congestion control, slow start, etc) are not implemented.
 
Specifics of the MCHP BSD socket interface implementation:
·                The MCHP BSD TCP/IP stack supports zero copy sockets. While the zero copy interface is not part of the BSD standard,
it has the advantage of providing a higher performance TCP/IP communication for the application that uses this functionality.
The application has to provide acknowledge callbacks for the TX flow and call an acknowledge function for the RX flow
when done processing the buffer.
·                The hardware provided RX checksum is now used for UDP/TCP checksum calculation.

Fixes

·                The current release is a major re-work of the version 1.0.01.
Many features have been added or modified and the performance has been improved.

Known Problems

·                Common conversion routines (htons, htonl, and the like) still have proprietary names.
·                SO_BROADCAST option for an UDP socket that allows sending the broadcast messages is missing.
However to send broadcast messages the socket must set the destination address to 0xffffffff; 
·                TCP flow control is not implemented:
o                     For RX traffic it does not dynamically change the size of the advertised window but it uses a fixed window size.
o                     For TX traffic it does not correlate the amount of data transmitted according to the window advertised by the remote party.
·                Nagle algorithm is not correctly implemented: it does not use the packets acknowledgement but rather uses a timer to send small queued packets.
·                The external PHY interrupts are not used.

Resources

A HTTP with DHCP client demo application was used with the build parameters:
    -g -Os -fomit-frame-pointer
 
The following results were obtained for the BSD stack (just the TCP/IP layer, no other higher level protocols):
·                Flash < 17 KB
·                Static RAM < 0.5 KB
·                Dynamic RAM is user configurable. A common setting across demo projects is:
o                     MAC RX buffers: 7.5 KB (tcpip_bsd_config.h:: EMAC_RX_DESCRIPTORS=10, EMAC_RX_BUFF_SIZE = 1536/2)
o                     Stack Manager local heap: 10 KB (tcpip_bsd_config.h:: BUFF_MGR_HEAP_SIZE = (1024*10)
